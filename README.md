# 电商平台商品详情页优化 A/B Test

## 📌 项目简介

本项目模拟某电商平台针对**商品详情页首屏改版**开展 A/B Test，通过统计学方法验证新版页面是否能够提升用户购买率，并结合实验结果给出业务决策建议。

项目基于公开数据集进行实验模拟，使用 **MySQL + Python** 完成数据清洗、实验分流、样本量计算、A/A Test、公平性验证、A/B Test、统计检验及结果分析。

---

## 🎯 项目背景

某电商平台在新春活动前发现用户购买率仅约 **12%**。漏斗分析显示，“浏览 → 加购”环节流失较高，因此产品团队计划优化商品详情页首屏，将商品卖点、用户评价、服务保障及促销信息前置展示，并强化 CTA（加入购物车、立即购买）按钮，希望将购买率提升至 **13%**。

实验目标：

* **H0（原假设）**：新版页面不会显著提升购买率。
* **H1（备择假设）**：新版页面会显著提升购买率。

## 🔄 项目流程

### 1. 数据清洗（SQL）

对原始实验数据进行预处理，包括：

* 修改数据库关键字字段名称；
* 删除缺失值与重复值；
* 清理异常日期数据，仅保留实验周期内数据；
* 删除同时进入多个实验组的污染用户；
* 构建 `control_1`、`control_2`、`treatment` 三个实验分组，为后续 A/A Test 与 A/B Test 做准备。

---

### 2. 实验设计

* 实验单元：`user_id`
* 分流比例：50% / 50%
* Baseline：12%
* 目标转化率：13%
* 最小可检测效应（MDE）：8.33%
* 显著性水平（α）：0.05
* 检验功效（Power）：0.80

通过 Power Analysis 计算得到：

> 每组最小样本量约 **17,164**。

---

### 3. 实验公平性验证

在正式开展 A/B Test 前，首先进行实验系统校验：

* 检查 `group` 与 `page_type` 是否一一对应；
* 执行 **A/A Test**，验证两组旧版页面不存在统计学显著差异；
* 执行 **SRM（Sample Ratio Mismatch）检验**，验证实验流量分配符合预期，不存在样本比例失衡。

---

### 4. A/B Test 分析

以购买率（Conversion Rate）作为核心分析指标，对实验组与对照组进行比较。

分析内容包括：

* 购买率计算；
* 均值与标准差统计；
* 转化率提升（Lift）分析；
* 双样本比例 Z 检验；
* 95% 置信区间估计；
* 实验结果可视化。

---

## 📊 实验结果

实验结果显示：

| 指标  | Control | Treatment |
| --- | ------- | --------- |
| 购买率 | 12.18%  | 12.13%    |

统计检验结果：

* Z Statistic：0.1321
* P-value：0.8949 (> 0.05)

说明新版页面与旧版页面在购买率上不存在统计学显著差异，未达到预期将购买率提升至 13% 的目标。

最终实验结论为：

> **Neutral（持平）** —— 不建议全量上线当前版本。

---

## 💡 业务洞察

虽然新版详情页优化了信息展示结构，但实验结果表明，仅调整页面排版不足以显著提升用户购买率。

后续建议：

* 强化优惠券、限时折扣等利益点刺激；
* 针对不同用户群体开展细分分析；
* 排查不同设备或渠道的异质性表现；
* 继续迭代页面设计并开展下一轮 A/B Test。
